# 115 MPV 原画点播 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在115官方视频页加入单次“MPV 原画”按钮，通过当前用户的 Windows 自定义协议启动本机 MPV，并暂停网页播放器。

**Architecture:** 用户脚本只从页面现有 `performance` 资源记录捕获 HTTPS HLS 地址，不调用115未公开接口。`mpv115://` 协议把编码后的 URL、标题和 User-Agent 交给一个只做验证与参数转发的 PowerShell 处理器。

**Tech Stack:** Tampermonkey JavaScript、Node.js `assert`、Windows PowerShell、HKCU 自定义 URL 协议、MPV。

## Global Constraints

- 仅手动单次点播；不批量、并发、自动连播或自动重试。
- 固定原画并暂停网页播放器。
- 不下载视频，不传递或持久化 Cookie、令牌及浏览器存储。
- MPV 固定为 `C:\Users\mugon\Documents\PythonStudio\MyMPV\mpv.com`。
- 不增加第三方依赖。

- [ ] **Step 3: Implement the minimal page flow**

Add the metadata include and call `init115MpvPlayback()` after `pageInit()`. The initializer must return immediately outside115 video pages. On the target page it uses one bounded `MutationObserver` to find the existing “原画” button and the adjacent actions, injects one button, and disconnects.

The click handler must:

```javascript
button.disabled = true;
const before = new Set(get115M3u8Entries());
originalButton.click();
const stream = await waitFor115M3u8(before, 5000);
if (!stream) throw new Error("未捕获到原画地址");
const target = "mpv115://play?" + new URLSearchParams({
    url: stream,
    title: document.querySelector("h1")?.textContent?.trim() || document.title,
    ua: navigator.userAgent
});
location.href = target;
const video = document.querySelector("video");
if (video) video.pause();
```

`get115M3u8Entries()` filters only valid HTTPS URLs containing `.m3u8`. `waitFor115M3u8()` first accepts a newly recorded URL, then falls back to the newest existing entry when the page is already on original quality. Errors use the existing `showAlert`; there is no retry or concurrent request.
