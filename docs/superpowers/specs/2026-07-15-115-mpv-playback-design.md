# 115 MPV 原画点播设计

## 目标

在 `https://115.com/players/video/*` 官方视频页增加“MPV 原画”按钮。用户单次点击后，脚本复用网页播放器已请求的原画 HLS 地址，通过 Windows 自定义协议启动本机 MPV，并暂停网页播放器。

## 范围

- 仅手动单次点播，不批量、并发、自动连播或后台轮询接口。
- 不下载媒体文件，不导出或传递115 Cookie、令牌及浏览器存储。
- 固定选择原画；失败时提示用户，不自动重试。
- MPV 路径固定为当前检测到的 `C:\Users\mugon\Documents\PythonStudio\MyMPV\mpv.com`。

## 用户脚本

现有 `javbus-larger-thumbnails.user.js` 增加115视频页匹配和一个独立初始化入口，不改变其他站点现有流程。

页面播放器出现后，在其操作区注入“MPV 原画”按钮。点击时：

1. 禁用按钮，防止重复点击。
2. 若网页视频默认暂停，调用一次页面 `<video>.play()`，让播放器生成正常点播请求。
3. 使用页面现有清晰度控件选择“原画”。
4. 从 `performance` 资源记录和随后出现的资源条目中选择本次切换产生的最新 HTTPS `.m3u8` 地址；不主动调用115播放接口。
5. 未在有限等待时间内取得地址时恢复按钮并显示错误。
6. 取得地址后组装 `mpv115://play?url=...&title=...&ua=...`，所有值均使用 URL 编码。
7. 触发协议处理器后调用当前页面 `<video>.pause()`，随后恢复按钮。

脚本只接受 `https:` 且路径或查询中包含 `.m3u8` 的地址。不会把 `blob:` 地址交给 MPV。

## Windows 协议处理器

仓库增加两个 PowerShell 文件：

- 安装脚本在 `HKCU\Software\Classes\mpv115` 注册自定义协议，无需管理员权限。
- 处理脚本解析协议 URI，验证动作是 `play`、媒体地址为 HTTPS，并将参数作为独立参数传给固定的 `mpv.com`。

MPV 参数包括115来源页、浏览器 User-Agent 和视频标题；不包含 Cookie。处理脚本不使用 `Invoke-Expression`、拼接命令行或启动 shell。

Chrome 首次调用可能显示外部应用确认；用户可选择始终允许。

## 错误处理与安全

- 页面结构或清晰度控件变化：显示“找不到原画控件”。
- 未发现原画流：显示“未捕获到原画地址”。
- 协议未安装：浏览器不会成功启动 MPV，按钮提示用户运行安装脚本。
- 本地处理器拒绝非 HTTPS、缺少 URL、错误动作及不存在的 MPV 路径。
- URL 和标题不写日志、不复制到剪贴板、不持久化。

## 验证

沿用仓库现有的无框架 Node `assert` 测试风格，先增加失败检查，再实现最小代码。测试覆盖：115页面匹配、按钮、原画选择、仅 HTTPS `.m3u8`、协议调用、暂停网页视频、不传 Cookie，以及安装器的当前用户注册位置。

完成后运行全部 `test-*.js`，并在115视频页手动验证一次：选择原画、启动 MPV、网页播放器暂停。手动验证不批量请求，也不修改账号数据。
